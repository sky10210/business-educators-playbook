# Content Schema

Every strategy Markdown file uses YAML front matter.

Required fields:
- title
- slug
- strategy_id
- type
- order
- status
- difficulty
- time_min
- time_max
- technology
- movement
- best_for
- courses
- grades
- problems_solved
- related
- seo_title
- seo_description
- tags

The body uses stable section headings so the website can later convert sections into styled components.
